# Generated Code

## Overview

WaveMaker gnerated code is **not black boxed.**. It generates **fully readable, standards-based Java source code** that follows proven enterprise architecture patterns.  
All backend artifacts are built using **Java, Spring, and Hibernate/JPA**, ensuring transparency, control, and long-term maintainability.



---



## Generated Backend Architecture

The following diagram illustrates the backend architecture generated by WaveMaker.

Client requests are handled by **REST controllers**, processed through a **service layer** that encapsulates business logic, and persisted via **DAO/repository components** backed by **Hibernate/JPA**.  
Entity models represent database tables, while **design-time metadata** supports development without affecting runtime execution.

This architecture follows standard **Java and Spring patterns**, ensuring transparency, extensibility, and enterprise readiness.

## Layered Architecture

### REST Controller Layer (`controller`)

**Role**  
Acts as the API entry point for client applications.

**Responsibilities**
- Expose RESTful endpoints
- Handle request–response lifecycle
- Perform authorization and input validation
- Marshal and unmarshal JSON payloads

**Implementation**
- Implemented using standard Spring REST controllers
- Easily extensible for custom endpoints or security logic

---

### Service Layer (`service`)

**Role**  
Encapsulates business logic and application workflows.

**Responsibilities**
- Implement business rules
- Validate inputs
- Manage transactions
- Coordinate interactions between controllers and data-access layers

**Implementation**
- Recommended layer for adding custom logic
- Isolated from regeneration during platform upgrades

---

### DAO / Repository Layer (`dao`)

**Role**  
Handles database interactions using ORM abstractions.

**Responsibilities**
- Execute persistence operations using Hibernate/JPA
- Abstract database-specific logic
- Provide a clean data-access interface to the service layer

**Generated Capabilities**
- Standard CRUD APIs per entity
- Filtering and pagination APIs
- Count and export APIs
- Support for **custom queries and stored procedures**

**Implementation**
- Uses standard JPA repositories
- Avoids proprietary or lock-in data-access abstractions

---

### Model Layer (`models` / Entity Classes)

**Role**  
Represents database schema as Java objects.

**Responsibilities**
- Define entity fields and relationships
- Map tables using JPA annotations
- Serve as data carriers across layers

**Implementation**
- Plain Old Java Objects (POJOs)
- Fully extensible and reusable

---

### Design-time Configuration (`designtime`)

**Role**  
Stores metadata required by WaveMaker Studio during application design.

**Contents**
- Database connection definitions
- API specifications
- Query and stored procedure metadata
- Schema information

**Notes**
- Used exclusively at design time
- Runtime execution depends only on generated Java source code
