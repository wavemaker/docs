# Variables and Types in WaveMaker

This document explains **what variables are in WaveMaker**, the **different variable types available**, and **when to use each one** while building applications.

---

## What is a Variable in WaveMaker?

A **variable** in WaveMaker is a configurable entity used to:

* Store data
* Fetch data from databases or services
* Bind data to UI components
* Trigger actions (create, update, delete, fetch)
* Maintain application or page state

Variables act as the **data and action layer** between the UI and backend services.

---

## Variable Scope

WaveMaker variables are scoped at different levels depending on where they are created:

### 1. App-Level Variables

* Available across the entire application
* Useful for shared data (logged-in user info, global settings)

### 2. Page-Level Variables

* Available only within a specific page
* Destroyed when the user navigates away from the page
* Ideal for page-specific data and actions

---

## Types of Variables in WaveMaker

WaveMaker provides multiple variable types to support different use cases.

---

## 1. Data Variables

**Purpose:**

* Interact with database tables

**Key Features:**

* Auto-generated CRUD operations
* Strongly typed based on database schema
* Supports pagination, filtering, and sorting

**Common Operations:**

* Read
* Insert
* Update
* Delete

**Use Case Example:**
Fetching a list of users from a database table and displaying it in a data table.

---

## 2. Service Variables

**Purpose:**

* Consume backend services

**Supported Service Types:**

* REST services
* Java services
* SOAP services
* Swagger / OpenAPI integrations

**Key Features:**

* Auto-generated request and response models
* Configurable headers, query params, and body
* Supports authentication and security configurations

**Use Case Example:**
Calling an external payment API or an internal Java service.

---

## 3. Live Variables

**Purpose:**

* Enable real-time data updates

**Key Features:**

* Automatically refresh data when backend changes
* Useful for dashboards and collaborative apps

**Use Case Example:**
Displaying live order status updates without page refresh.

---

## 4. Model Variables

**Purpose:**

* Hold temporary or structured data

**Key Features:**

* User-defined data models
* Can represent single objects or lists
* Does not automatically connect to backend

**Use Case Example:**
Storing form data before submitting it to a service.

---

## 5. App Variables

**Purpose:**

* Maintain global state across pages

**Key Features:**

* Accessible from any page
* Persists during the app session

**Use Case Example:**
Storing logged-in user details or selected language.

---

## 6. Page Variables

**Purpose:**

* Manage page-specific data

**Key Features:**

* Exists only during the lifecycle of a page
* Helps reduce memory usage

**Use Case Example:**
Holding selected row data in a page-level table.

---

## 7. Device Variables

**Purpose:**

* Access device-specific capabilities

**Key Features:**

* Camera
* GPS
* File upload
* Contacts (based on platform support)

**Use Case Example:**
Capturing an image using a mobile device camera.

---

## Variable Data Types

WaveMaker variables support multiple data types, including:

* String
* Number (Integer, Long, Double)
* Boolean
* Date / Timestamp
* Object
* List / Array

These types are either:

* Auto-derived (from database or service definitions)
* Manually defined (in model variables)

---

## Data Binding with Variables

Variables are tightly integrated with WaveMaker’s UI:

* Bind variable data directly to widgets
* Trigger variables on events (onClick, onLoad, onChange)
* Chain variables using success and error callbacks

---

## Best Practices

* Use **Page Variables** whenever data is page-specific
* Prefer **Model Variables** for form handling
* Avoid overusing **App Variables** to prevent unwanted coupling
* Name variables clearly based on their purpose
* Handle error and success callbacks for service and data variables

---

## Summary

Variables are the backbone of data flow in WaveMaker applications. Understanding the different variable types and scopes helps in building scalable, maintainable, and efficient applications.

Choosing the right variable type ensures better performance, cleaner design, and easier debugging.
