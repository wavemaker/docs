# Overview

### What are UI Events? 

Events are actions or occurrences in the browser that your code can respond to. They allow applications to react dynamically to changes without requiring components to constantly monitor for updates.

#### Event Types
Events can be grouped by the type of user interaction or browser action that triggers them:
- **Mouse Events:** click, double click, mouse enter, mouse leave.

```javascript
Page.UserButtonDblclick = function ($event, widget) {
    alert('The User button double click event has been triggered');
};
```

- **Keyboard Events:** keypress, keydown, keyup.
```javascript
Page.UserButtonKeypress = function ($event, widget, item, currentItemWidgets) {
    alert('The User button key press event has been triggered');
};
```
- **Form Events:** submit, change, focus, blur, input.
```javascript
Page.employeeFormSubmit = function ($event, widget, $formData) {
    alert('The employeeForm submit event has been triggered');
};
```

- **Window/Document Events:** load, unload, resize, scroll, DOMContentLoaded. 
```javascript
window.onload = function () {
    alert('Window loaded successfully');
};
```

#### Event Object
Every event handler function, a parameter is often defined with names such as ```event``` , ```evt``` , or ```e```. This parameter is referred to as the event object. It is automatically passed to event handlers and provides additional information and functionality related to the event that was triggered. This object contains useful properties and methods related to the event
- **event.type:** Returns the type of the event as a string (e.g., 'click', 'keydown').
- **event.target:** Refers to the element that triggered the event.
- **event.preventDefault():** A method to stop the browser's default action for that event (e.g., preventing a form from submitting). 

## UI Event Handling in WaveMaker

WaveMaker provides a robust event driven architecture that allows developers to execute custom logic at different stages of the application lifecycle. 

**Events are broadly categorized into**
- App-level
- Page-level
- Component-level.
