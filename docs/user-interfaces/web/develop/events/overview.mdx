# Overview

## What are Events?

An **event** is a notification generated by a system to indicate that a specific action, interaction, or state change has occurred. Events enable applications to respond dynamically to changes without requiring components to constantly check for those changes.

### How Events Are Emitted

Event emission is the process of generating and broadcasting an event when a specific action or condition occurs.

**Events can be emitted by:**
- User interactions (click, keypress, selection)
- System actions (data fetch, API response)
- State changes (variable value updated)
- Lifecycle hooks (component initialized, destroyed)

### How Events Are Subscribed

Event subscription is the mechanism by which code listens for and responds to events.

**Subscribing involves:**
- Identifying the event source
- Registering a handler (callback or function)
- Executing logic when the event fires

## Event Handling in WaveMaker

WaveMaker supports event-driven behavior at two levels:
- Variable level event handling
- UI Components level event handling

Both follow the same fundamental event model but differ in purpose and scope.
