WaveMaker 11.15.2 brings a focused set of enhancements to improve styling flexibility, widget behavior, and overall developer experience.
It also delivers key stability fixes across UI, theming, and platform behavior, ensuring more consistent and reliable app performance.


## Enhancements

### React Native

<details><summary>Floating Label Styling and Behavior Improvements</summary>
Fixes issues with floating labels not behaving correctly and enables better styling control through the Design Style Palette.
</details>

<details><summary>Donut Chart: Text Color Customization</summary>
Adds support to edit and apply custom text color codes for Donut chart labels.
</details>

<details><summary>Wizard Widget: Async Handling in onNext</summary>
Ensures the Wizard waits for API responses or async operations in the `onNext` callback before proceeding. Returning `false` prevents navigation to the next step.
</details>

<details><summary>WMX Component Visibility Control</summary>
Ensures WMX components are properly hidden when the `show` property is set to `false`.
</details>

<details><summary>Preview Stability After Adding WMX Components</summary>
Resolves preview failures caused by React version mismatches and missing dependencies when WMX components are added.
</details>

<details><summary>Form Field Keyboard Event Support</summary>
Adds support for `onKeypress` and `onSubmitEditing` events on Text, Number, and Currency fields. Textarea supports `onKeypress` only.
</details>

<details><summary>Custom Global Error Handling Support</summary>
Introduces a toggle to disable default error UI and allows handling errors using `App.onError` for custom navigation or UI.
</details>

<details><summary>Floating Label Default Position Fix</summary>
Ensures floating labels remain positioned at the top even when input fields are empty.
</details>

<details><summary>Chart Content Height Configuration</summary>
Adds a `chartheight` property to control the height of the chart plotting area independently of the container.
</details>

<details><summary>Zero Value Visibility in Bar and Column Charts</summary>
Adds a `showzerothvalue` property to visually distinguish zero-value data points using an outline indicator.
</details>


## Bug Fixes

### React Native

<details><summary>Background Image Rendering in Container</summary> Fixes inconsistent background image rendering between preview and emulator environments, ensuring images correctly cover the container. </details> <details><summary>Badge Styling and Token Support</summary>

Fixes an issue where badge styling tokens (badge.background-color, badge.color, badge.margin-left) were not reflected in preview.

Adds new tokens for better customization:

badge.height
badge.width
badge.font.size
badge.line-height
badge.border.radius

For circular badges, border.radius should be set to half of height/width.

</details> <details><summary>Message Text Color Token Exposure</summary> Exposes message text color tokens in the Style Workspace for better customization. </details> <details><summary>Input Field Font Weight in Focus State</summary> Fixes inability to customize font-weight for input fields in the focused state. </details> <details><summary>Select Component Styling and Variant Issues</summary>

Fixes multiple issues with Select component styling:

Variant styles not applying correctly in disabled state
Select-specific variants not appearing in variant selection
Arrow button styling (border, padding, width) not configurable
</details> <details><summary>Text Decoration Support for Anchor and Popover</summary>

Fixes issues where text-decoration styles were not applied correctly to Anchor components and were missing for Popover components in the Style Workspace.

</details> <details><summary>Bottom Sheet Styling Controls</summary> Fixes inability to remove drag handle and customize width from Style Workspace and widget properties. </details> <details><summary>Input Field Focus Styling Override</summary> Fixes inability to remove default focus border styling for input fields due to missing design tokens. </details> <details><summary>Anchor Text Decoration Not Applying</summary> Resolves issue where `text-decoration: none` was not applied correctly, causing underline to persist. </details> <details><summary>Popover Token Coverage</summary> Fixes missing and non-functional tokens for Popover widget styling in the Style Workspace. </details> <details><summary>Variants Not Reflecting in Expo Preview</summary> Fixes issue where variants created during design time were not reflected in Expo preview. </details> <details><summary>Styling Issues After Migration</summary>

Resolves UI inconsistencies observed after migrating projects:

Double borders in Bottom Sheet components
Unexpected underline or duplicate borders in input and display fields
</details> <details><summary>Padding with Multiple Values</summary>

Fixes issue where multi-value padding was not applied correctly.

For example:

Horizontal padding applied correctly
Vertical padding (top/bottom) failed when multiple values were provided

Now supports proper application of both horizontal and vertical padding values.

</details> <details><summary>Dialog Overlay Styling Exposure</summary> Exposes overlay background styling properties for the Dialog component in the Style Workspace. </details> <details><summary>Accordion Styling Properties Exposure</summary> Adds support for configuring title background and individual accordion border styles. </details> <details><summary>Currency Widget Token Support</summary> Adds missing design tokens to enable styling of the Currency widget. </details> <details><summary>Tab Widget Token Support</summary>

Adds missing tokens for Tab widget customization:

Padding and bottom border for tab headers
Margin for active tab indicator
Font family for tab header text
</details> <details><summary>Accordion Pane Styling</summary> Exposes pane-level styling for Accordion component through Style Workspace. </details> <details><summary>Button Border Width Per Side</summary>

Fixes inability to apply border width individually per side.

Previously, border styles only worked when applied uniformly. Now supports independent control for each side.

</details> <details><summary>Tabbar Styling Improvements</summary>

Fixes multiple styling limitations in Tabbar:

Height not applying correctly
Border width per side not configurable
Padding and margin for icons and tab items not adjustable
</details> <details><summary>Floating Input Placeholder Visibility</summary> Fixes issue where placeholder text was not visible in floating input fields. </details> <details><summary>Container Show/Hide Behavior</summary> Fixes inconsistent show/hide behavior of containers in preview. </details> <details><summary>Select Component Arrow Button Styling</summary>

Exposes styling controls for Select component arrow button, including:

Border properties
Height and padding
Removal of default styles
</details> <details><summary>Form Field Border and Disabled State Styling</summary>

Fixes issues where:

Border width updates were not applied
Disabled background color did not reflect changes
Borders remained visible even when set to 0
</details> <details><summary>Modal Dialog Padding</summary> Fixes inability to override default dialog padding (previously fixed at 24px). </details> <details><summary>Heading Default Margin Override</summary> Fixes issue where default 4px margin persisted despite Style Workspace overrides. </details> <details><summary>Button Text Alignment and Icon Styling</summary>

Fixes issues where:

Text alignment settings were not applied correctly
Button text remained left-aligned
Icon color could not be styled independently from text
</details> <details><summary>RadioSet Scrolling on iOS</summary>

Fixes unintended scrolling of RadioSet inside List widget on iOS.
Behavior is now consistent across both iOS and Android.

</details> <details><summary>Dropdown Position with Progress Circle Header</summary> Fixes incorrect dropdown positioning when a Progress Circle Header is present on the page. </details> <details><summary>Container Padding and Token Support</summary>

Fixes container padding not applying correctly and adds missing tokens for:

Padding (all sides)
Border width (individual sides)
</details> <details><summary>Autocapitalize Support in Web Preview</summary>

Adds proper auto-capitalization support:

Words mode capitalizes each word
Sentences mode capitalizes after punctuation
</details> <details><summary>Date Widget Selection Issue</summary>

Fixes issues where:

Calendar buttons were not working
Selected date did not match displayed value
</details> <details><summary>List Scroll Behavior in Tabs</summary>

Fixes issue where full page scrolled instead of restricting scroll within the List widget inside a tab pane.

</details> <details><summary>Dynamic Autocapitalize Binding</summary> Fixes issue where dynamically bound autocapitalize values were not applied and defaulted to null. </details> <details><summary>Alert Dialog Styling Tokens</summary> Adds missing styling support for Alert Dialog in Design System Workspace. </details> <details><summary>Dark Theme Token Application</summary>

Fixes issue where updated token values were not reflected in Dark theme, ensuring consistency across themes.

</details> <details><summary>UI Freeze on Rapid TabBar Interaction</summary>

Fixes UI freeze caused by rapid TabBar interactions.

Introduces disableNavigationAnimation configuration to prevent UI thread blocking and ensure smooth navigation.

</details> <details><summary>CheckboxSet Accessibility in Dialog</summary>

Fixes accessibility issues where CheckboxSet items inside Dialog were not visible in Appium Inspector.

Individual checkbox elements and labels are now accessible for testing and automation.

</details>
